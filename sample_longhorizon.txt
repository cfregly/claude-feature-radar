
  Long-horizon benchmark on Claude Haiku 4.5. Two runs of a 24-report chain,
  each report about 22,000 tokens. The unbounded run is expected to lose
  the thread (a wrong answer) or, on heavier payloads, exceed the 200,000
  window. Estimated about $1 to $2 and a couple of minutes on Haiku. The measured cost and
  time are printed and saved when it finishes.

  Long-horizon chain audit on Claude Haiku 4.5. Caching ON in both runs.
  Workload: a chain of 24 incident reports, each about 22,000 tokens (a
  realistic large tool payload). The agent reads them one at a time, in order, and counts
  the URGENT ones. The only difference between the runs is Claude's managed context:
  context editing (bound the window) plus the memory tool (keep the count across clears).
  Claude Haiku 4.5 context window: 200,000 tokens. True URGENT count: 8.

  WHAT YOU GET
    Unbounded agent : WRONG answer 2 (the true count is 8). It lost the
                      thread once context passed ~153k tokens, gave up after about
                      9 of 24 reports, and miscounted. A silent failure you ship.
    Context-edited  : answer 8 (correct). Held context flat at ~39k
                      tokens, read all 24 reports, counted via the memory file. $1.1066, 52s.

    The value, measured: the SAME model gets this long job WRONG on its own (2) and
    RIGHT with Claude's managed context (8), for $1.1066.

  WHY IT FAILS UNMANAGED
    The unbounded arm's carried context climbed 687 -> 153,957 tokens
    and its per-turn cost climbed 34x (0.00107 -> 0.03577).
    The edited arm held context near 39,564 tokens with a flat per-turn cost.
    Pushed harder (a longer chain or bigger payloads) the unbounded arm stops failing
    silently and fails loudly: it exceeds the 200,000 window and the API errors.

  HONEST COST: the edited run cost $1.1066 total, MORE than the unbounded
  run's $0.2591, because the unbounded run quit early (and wrong) and
  because clearing rewrites the cached prefix. The win here is a correct answer and a bounded
  context, not a cheaper bill. On a long enough job the unbounded arm cannot finish at all.

  Reproduce: `make longhorizon` on Claude Haiku 4.5, your own key. This run cost $1.3656 total
  and took 69s wall.
