  Long-horizon chain audit on Claude Haiku 4.5. The isolated value of context editing.
  Workload: a chain of 8 incident reports, each about 40,000 tokens (a
  realistic large tool payload, e.g. a big log or trace). The agent reads them one at a
  time and counts the URGENT ones.
  ISOLATION: both runs send the model the same request, same model, same prompt, the MEMORY
  TOOL ON in both, caching ON in both. The only thing that changes what the model receives is
  context editing, so any difference below is caused by context editing alone. (The editing-
  off arm also records its expected window error as a measured event, which the model never
  sees.) Claude Haiku 4.5 window: 200,000 tokens. True count: 3.

  WHAT YOU GET
    Editing OFF : CRASHED at step 8. Input 203,056 tokens exceeded the
                  200,000 window, the API rejected the request, the run cannot finish.
    Editing ON  : finished all 8 reports, answer 3 (correct), context held flat
                  at ~34k tokens. $0.35, 19s.

    The value, measured and isolated: context editing is what lets the long run FINISH.
    Identical setup, one flag. Without it the API errors at the window; with it the agent
    completes. That is a reliability win caused by context editing, not by the memory tool.

  WHY IT FAILS WITHOUT EDITING
    The editing-OFF context climbed 1,816 -> 187,471 tokens and its
    per-turn cost climbed 24.7x (0.00225 -> 0.05567). The
    editing-ON arm held context near 34,256 tokens at a flat per-turn cost.

  WHERE CORRECTNESS COMES FROM (the honest split)
    The MEMORY TOOL (on in both arms) is what makes the count correct: the agent tallies into
    a durable file instead of an in-context count. CONTEXT EDITING is what makes the long run
    survivable: it bounds the window so the agent never hits the wall. Two primitives, two
    jobs. This benchmark isolates context editing only.

  HONEST COST: context editing is not a cheaper bill. Clearing rewrites the cached prefix,
  so on a job short enough to finish either way the editing-ON run can cost MORE. Its value
  is that a heavy job finishes at all. The editing-ON run here cost $0.35.

  Reproduce: `make longhorizon` on Claude Haiku 4.5, your own API key. This run cost $0.64 total
  and took 36s wall.

  RELIABILITY (3 runs of the same setup, only context editing toggled)
    editing OFF: 3/3 FAILED (2 crashed at the window, 1 returned a wrong answer), 0/3 correct.
    editing ON : 3/3 finished with the correct answer, context bounded near ~34k tokens.
    The failure mode varies (a crash or a wrong answer), but unbounded context failed every
    run pushed this far, and context editing succeeded every run. Small N, stated plainly.
