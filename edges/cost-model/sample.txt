  === Edge 1: 1h cache TTL, no per-hour storage fee (bursty-recurring traffic) ===

  WIN regime: a 50k-token prefix reused 12 times over 8h, reads ~40 min apart (gaps exceed OpenAI best-effort eviction)
    Claude   write $0.3000 + reads $0.1800 + storage $0.0000  = $0.4800
    Gemini   write $0.1000 + reads $0.1200 + storage $1.8000  = $2.0200  (per-hour storage meter)
    OpenAI   reads $0.2625  = $0.2625  (gaps fit the cache window, reads hit at the cached rate)
    -> Claude wins vs Gemini storage: True; vs OpenAI eviction: False
    why: Claude's flat 1h write plus 0.1x reads with NO storage beats Gemini's per-hour storage meter, and beats OpenAI's best-effort cache which evicts between the sparse bursts and reprocesses at full input price

  LOSE regime (high QPS): the same prefix reused 2000 times, reads seconds apart (continuous reuse)
    read-line bill  Claude $30.0000  Gemini $20.0000  OpenAI $25.0000
    -> Claude loses: True  (the 0.1x read multiplier is identical across all three vendors, so the bill collapses toward base input price, where Claude is the most expensive)

  LOSE regime (long hold): a prefix that must stay warm 3h between reads (longer than Claude's 1h max TTL)
    Claude (1h TTL expires) $0.6000  vs OpenAI 24h extended $0.1625
    -> Claude loses: True  (Claude's cache TTL maxes at 1h, so a 3h hold expires and reprocesses at full input price, where OpenAI's 24h extended retention still hits the cache)

  Crossover: Claude wins below roughly one read every 60 minutes held under 1 hour (sparse bursty reuse, where the no-storage-fee flat cache beats Gemini's storage meter and OpenAI's eviction); Claude loses above that density (high QPS collapses to base price) and beyond a 1-hour hold (OpenAI 24h retention)

  === Edge 2: no long-context premium (flat 1M-window input band) ===

  WIN regime: a ~300k-token per-call context (above the 200k surcharge band)
    Claude input $0.9000  vs Gemini 3.1 Pro input $1.2000
    -> Claude wins: True  (Claude bills the full 1M window flat at $3.00/MTok, where Gemini 3.1 Pro charges $4.00/MTok above 200k (double its sub-200k rate))

  LOSE regime: the same prompt at ~50k tokens (below the 200k threshold)
    Claude input $0.1500  vs Gemini 3.1 Pro input $0.1000
    -> Claude loses: True  (below the threshold there is no surcharge edge, and on raw per-token input price Gemini 3.1 Pro ($2.00/MTok) is cheaper than Claude (Claude Sonnet 4.6, $3.00/MTok))

  Crossover: Claude wins above the 200k-token surcharge threshold (flat band vs the competitor's doubled above-200k rate); Claude loses below it (no surcharge edge, and the competitor's raw input price is lower)
